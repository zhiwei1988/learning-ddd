## C++ 教学策略（有效，继续用）
- 第 4 课验证了一个好路子：**先把 DDD 概念挂到用户已有的 C++ 肌肉记忆上**
  （「你早就在写值对象了，只是不知道它叫这名字」），再教「什么时候不该这么写」。
- **C++ 代码必须真编译**。验证脚本在 **`code/`**（见 `code/README.md` 的跑法），不要放 scratchpad——
  那是 session-specific 的，换会话就没了。第 4 课示例已在 clang 17 / `-std=c++20 -Wall -Wextra` 下
  编译 + 跑 8 条断言（含 const 成员删除拷贝赋值的 static_assert）。
  首次编译就抓到两个错（assert 宏吃不下花括号初始化列表、漏 include <type_traits>）——不编译就发出去会很难看。
- 贴**真实的编译器报错**比描述有效得多（第 4 课贴了 const 成员那个坑的原始 clang 报错）。

## 组件库（assets/）
- `course.css` — 全课共享样式。含 `.compare`（左右代码对照）、`.card`、`.ctx-map/.ctx-box`
  （上下文/子域方框图，配 `.cols-2` `.cols-3`）、`.zh`（引文里的中文译文取消伪斜体——中文无真斜体）。
- `quiz.js` — `renderQuiz(root, questions, opts)`，opts.perfect/imperfect 覆盖结语文案。
- `categorize.js` — `renderCategorize(root, {buckets, items, perfect, imperfect})`，分类练习。
- 教训：结语文案曾硬编码在组件里，第 3 课复用时弹出了第 2 课的文案。**新增组件一律把课程专属文案做成参数。**
- 验证方式：`python3 -m http.server 8731` + claude-in-chrome（扩展不能导航 file:// URL；
  且 JS/CSS 会被浏览器缓存，验证改动要加 `?cachebust=N`）。
- **浏览器窗口会塌成 0 宽**：此时截图超时、`clientWidth` 为 0、所有元素都被误判成横向溢出。
  遇到 overflowX 异常先查 `document.documentElement.clientWidth`——是 0 就是假警报，
  开个新标签页（tabs_create_mcp）重测，resize_window 救不回来。
