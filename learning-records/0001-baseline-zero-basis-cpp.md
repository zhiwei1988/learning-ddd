# 基线：零基础起步的 C++ 工程师

用户自述 DDD 几乎零基础（说不清战略/战术设计、聚合、限界上下文），主力技术栈为 C++。动机为多重：新项目要用 DDD 落地、向架构师方向发展、系统性补课——但**没有**遗留系统改造的诉求。

## Implications

- 最近发展区从最底层概念开始：统一语言（0001 课）→ 限界上下文 → 子域，再进战术模式。
- 所有代码示例用现代 C++；讲值对象时可借力 C++ 值语义（拷贝、不可变、`operator==`），这是用户已有直觉，能降低认知负担。
- 无遗留系统包袱，示例可用干净的绿地建模，不必教「防腐层包老系统」这类改造技巧（推迟到有需要时）。
