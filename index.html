战略设计 · 系统与团队的尺度

01 统一语言 让代码说业务的话。消灭「需求 → 脑内翻译 → 实现」这个损耗层。 4/4
02 限界上下文 一个词，两个意思。为什么「全公司一个大模型」注定失败。 ✓
03 子域：核心 / 支撑 / 通用 把设计精力花在刀刃上——以及哪些地方根本不该用 DDD。 全对

战术设计 · 单个限界上下文内部

04 实体 vs 值对象 什么叫「同一个东西」。C++ 值语义的主场——你早就在写值对象了。 ✓
05 聚合：一次改动该覆盖多大范围 战术设计最难也最值钱的一块。边界 = 一致性边界 = 事务边界。 ✓
06 领域事件 边界外面的事怎么办？让「已发生的事实」成为一等公民——过去式、不可变、携 id。 全对
07 仓储与分层架构 依赖只朝里指。仓储是那道单向阀，编排住在应用层——你已经在用六边形架构了。 ✓
08 上下文映射与防腐层 镜头拉回战略：上下文之间怎么协作、怎么翻译，怎么用防腐层挡住上游烂模型的污染。 全对
09 领域服务 这段逻辑到底住哪：聚合方法 / 领域服务 / 应用服务。战术工具箱收官。 全对
